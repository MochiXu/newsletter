## 0. 动机:发现"准确率不高"是规律,不是 bug

连续运行一个月后观察到:对纳指 / 黄金 / 广义美元 / 2Y 这类**最深、最有效**的流动性资产,
**短期(1d/5d)方向预测命中率接近抛硬币**。这不是"技术指标 + AI"做得不够好——**这是弱式有效市场的常态**
(顶级量化在日线方向上胜率也就 51–53%,赚钱靠赔率/分散/校准,不靠"猜得准")。

→ **结论:不把"方向命中率"当优化目标**(追不到;追到了多半是过拟合或记忆污染)。
v1.6 改追三件**做得到**的事:
1. **架构更专业**:引入量化因子层(业内标准做法的最小版)。
2. **诚实的标尺**:能回答"我们到底有没有**技能**(打不打得过朴素基线)、**信心标得准不准**(校准)"。
3. **抓真能预测的东西**:因子层里做**波动率预测**——方向不可预测,但波动率有聚集性、可预测。

> 这与 [DESIGN.md](../../DESIGN.md) 的定位一致:卖的是**研究框架**,不是信号。一个诚实显示
> "短期方向不可预测"的 track record,对学习型用户反而是最有价值的输出。

## 1. 概念澄清(落地前先对齐三个词)

### 1.1 baseline(基线)= 必须打败的"零信息傻瓜规则"

只报"命中率 > 50%"会自欺,因为资产有漂移。v1.6 对每个 (资产 × horizon) 算三条基线:

| 基线 | 规则 | 为什么需要 |
|---|---|---|
| **漂移 drift** | 永远预测**历史多数方向**(美股长期上行 → "永远看涨")| 纳指日线 ~53–54% 天数在涨,5d/20d 更高;**AI 若 52% 其实比傻瓜还差** |
| **动量 momentum** | 预测方向 = 过去 N 日收益的符号("涨了继续看涨")| 趋势资产里常意外地强 |
| **因子 factor** | 预测方向 = 因子合成打分的符号(见 §2)| 我们**自己的量化模型**,AI 连它都打不过就该质疑 AI 这层 |

> **技能(skill)= AI 命中率 − max(三条基线)**。只有正技能才叫有预测价值。
> (扎心的真实结果:Meese-Rogoff 1983 发现汇率模型样本外**打不过"明天=今天"的随机游走**,四十年仍大致成立。)

### 1.2 校准(calibration)= 拿"AI 自报信心"对账"事后真实频率"

用户的核心疑问:"信心是 AI 随便给的,那**实际命中信心**是啥?"
**答:校准不需要算一个"真信心"。** 做法是**分桶**——把 AI 报 ~70% 的预测拢一桶,看这桶**实际命中了百分之几**:

| AI 自报信心桶 | 条数 n | 实际命中率(= 你问的"实际命中信心")| 判读 |
|---|---|---|---|
| 50–60% | … | … | 接近则校准好 |
| 60–75% | … | … | 实际<自报 = 过度自信 |
| 75%+ | … | … | AI 几乎不给极端值,样本会很少 |

"**实际命中信心**" = **某信心桶里的真实命中频率**,是**一整桶的统计量**,不是单条预测的数。

### 1.3 Brier score = 把校准压成一个数

**概率预测的均方误差**。例:AI 报"涨的概率 0.7" →
- 真涨(=1):(0.7−1)² = **0.09**;真跌(=0):(0.7−0)² = **0.49**。取所有预测平均,**越低越好**。
- 基线 Brier = 永远报历史涨跌比例。AI 的 Brier 得比它低才有用。

**对数损失(log loss)本期不做**:它专罚"报 90%+ 却错"的极端过度自信,而 AI 几乎不给 90%,用不上(用户已确认)。
**信息系数(IC)本期不做**:需连续打分 + 大样本,等因子层(连续打分)和数据攒够再上(留 V2/v3)。

## 2. 量化因子 / 信号层(`factors.py` —— 新架构层)

### 2.1 位置:夹在 features 和 llm 之间

```
features(原子技术特征:距MA / 收益率 / 已实现波动 / z / 52周分位 / 跨资产相关 …)
      │
      ▼
factors(新层:把原子特征聚合成"可解释因子打分",每资产标准化到可比尺度)
      │
      ├──▶ 喂给 LLM 当锚(prompt 特征块里多一节"因子打分",防瞎编、有依据)
      ├──▶ 序列化进 briefs.json 的 factors 块(前端可展示,与 signals 平级)
      └──▶ 自产「因子基线预测」+「代码信心」→ 记进 predictions.csv(§3 当基线打分)
```

- **不改骨架**:仍是"代码算数字、LLM 只解释"。factors 是 features 的**高阶聚合**,纯代码、因果。
- **单一事实源**:复用 `features.FEATURE_VIEW` / `snapshot_at` 的同一份 `snap`,避免与 signals 漂移。

### 2.2 因子定义(每资产一组,标准化为 z 或 [-1,1])

| 因子 | 来源(原子特征聚合)| 方向含义 |
|---|---|---|
| **趋势 trend** | 距 MA50 / 距 MA200 / MA 斜率 | 趋势跟随(正=多头排列)|
| **动量 momentum** | 5d / 20d / 60d 收益的标准化合成 | 趋势跟随 |
| **价值 value** | 52 周分位 / 距长期均值 | **均值回归**(极端=反向倾向)|
| **套息 carry** | 利率水平 / 实际利率 / 跨资产息差(资产相关)| 资产相关 |
| **波动 vol** | 已实现波动 z **+ 波动预测**(见 §2.4)| 风险尺(非方向)|
| **regime / breadth** | 复用 `regime.derive` 的代码派生标签 | 背景/条件 |

### 2.3 因子合成 → 方向基线 + 代码信心

- **合成方向(directional composite)**:先只用**有方向含义**的因子(趋势 + 动量为主,价值给小的反向权重):
  `score = w_trend·trend + w_mom·momentum − w_val·clip(value 极端)`。
  方向 = `sign(score)`(死区内记 flat)。**价值/套息/波动作为上下文喂 LLM,初版不强行进方向合成**(避免乱炖)。
- **代码信心(factor confidence)**:由 `|score|` 经单调映射到概率(初版 `0.5 + k·clip(|score|)` 封顶)。
  **初版同样未校准**——但它会和 AI 信心**一起进 §3 校准**,谁更校准谁可信。权重 `w_*`/`k` 先用合理默认,
  后续按 §3 校准结果回调(**避免过拟合:权重宜少、宜稳,不做花哨优化**)。
- **价值**:这条因子基线就是 §1.1 的第三条基线,也是 AI 的"陪练标尺"。

### 2.4 波动率预测(唯一"真能预测"的目标)

- 方向不可预测,**波动率有聚集性、可预测**。初版用 **EWMA(RiskMetrics λ≈0.94)** 对平方收益做指数加权,
  给出未来 h 日的年化波动预测;输出进 vol 因子 + factors 块。
- 价值:① 给简报真正"有 edge"的量化结论(风险升降);② 未来可单独打分(波动预测 vs 已实现波动——
  **这里才可能看到正技能**),为 v1.6/V2 的评估提供一个"对照组:难预测的方向 vs 可预测的波动"。

### 2.5 契约:新增 `factors` 块(向后兼容)

```jsonc
"factors": {
  "NASDAQCOM": {
    "scores": { "trend": 0.8, "momentum": 0.6, "value": -0.3, "carry": null, "vol": 1.2 },
    "composite": 0.62, "baselineDir": "up", "baselineConf": 0.61,
    "volForecastAnn": 0.18           // 年化波动预测
  },
  "XAUUSD": { ... }, "DTWEXBGS": { ... }, "DGS2": { ... }
}
```
- 脊柱级(模型无关,代码算一次,所有模型共享),与 `signals`/`regime` 平级。
- 前端可后续渲染成"因子打分条 + 波动预测";旧前端忽略未知字段即可。

## 3. 预测信心校准评估(前向、轻量 —— `evaluate.py`)

> **范围纪律**:只用**前向真实积累**的 `predictions.csv`(上线后逐日产、到期代码结算)。
> **不做回填**——回填有记忆污染(见 [v2 §1b](v2-progress.md)),会让评估测的是"记忆"不是"预测"。
> 完整的回填语料 / block bootstrap / 多年统计 / GDELT 仍归 V2-S2/S3。

### 3.1 信心语义钉死(校准的前提)

现状:`confidence` 是 AI 自评、无明确语义、未约束("随便给")。校准前必须**钉死含义**:

- **定义**:`confidence = P(预测方向在 horizon 内成立)` 的主观概率。**标度维持 0~1**(与现 `llm/schema.py`
  字段 + `models._coerce_conf` 一致;前端显示为百分比)——这次只**钉死语义**,不改标度,避免引入漂移。
- **落地**:把语义写进 `llm/schema.py` 的 `confidence` 描述 + `prompt.SYSTEM`;**不强行约束取值**(让 AI 自然铺开),
  但评估用**粗桶**(数据少:`<0.6 / 0.6–0.75 / 0.75+` 三桶)防止每桶样本过稀。
- 前端信心 tooltip(V1.5 已有"未校准、非真实概率")**补一句**:校准结果见命中率页(待接入)。

### 3.2 评估模块产出

`evaluate.py` 读 `predictions.csv`(只取 `status=settled` 的行),对 **每个模型 + 因子基线**算:

| 指标 | 计算 | 对应概念 |
|---|---|---|
| 命中率 hit | 逐 (模型 × 资产 × horizon) | 准确率(单看会自欺)|
| 三条基线命中率 | drift(历史多数方向)/ momentum(近 N 日符号)/ factor(§2.3) | §1.1 |
| **技能 skill** | `hit − max(基线)` | **核心结论** |
| **校准桶** | 按 §3.1 三桶,各桶 `{n, 自报均值, 实际命中率}` | §1.2 |
| **Brier** | 信心(0~1)与 0/1 结果的均方误差 + 基线 Brier | §1.3 |

- **drift / momentum 基线**:结算时从价格历史**确定性重算**(无需提前存)。
- **factor 基线**:**point-in-time,必须创建时记**(到期才能重算但代价高)→ §3.3 给 `predictions.csv` 加列。
- 输出 `data/scorecard.json`(前端命中率页未来读)+ 人读 `data/scorecard.md`。**前向 only**,带 `source:"forward"`。

```jsonc
// data/scorecard.json(草案,镜像 V2 track.json 的诚实字段)
{
  "asOf": "2026-06-23", "source": "forward", "primaryHorizon": "h_5d",
  "models": {
    "deepseek": {
      "byAsset": { "NASDAQCOM": { "h_5d": { "n": 12, "hit": 0.50,
                    "driftBaseline": 0.58, "momentumBaseline": 0.50, "factorBaseline": 0.55,
                    "skill": -0.08, "brier": 0.27, "brierBaseline": 0.25 } } },
      "calibration": [ { "lo": 0.5, "hi": 0.6, "n": 20, "stated": 0.55, "realized": 0.52 } ]
    },
    "_factor": { "byAsset": { ... } }     // 因子模型当一个"参赛选手"同样打分
  }
}
```

### 3.3 `predictions.csv` 增列(记 point-in-time 因子基线)

现 FIELDS `[created_date, model, asset, direction, horizon, confidence, status, resolved_date, realized_dir, realized_text, hit, note]`,**增**:
- `base_dir`(创建时因子基线方向)、`base_conf`(因子基线信心)。
- 幂等键不变 (created_date, model, asset, horizon);老行缺这两列时评估侧容错(跳过 factor 基线、仍算 drift/momentum)。
- **`predictions.csv` 只存原始事实**(预测 + 实现 + 基线快照);所有派生指标(技能/校准/Brier)由 `evaluate.py` 现算进 scorecard,不回写账本。

### 3.4 预期管理(诚实底线)

- **数据会很薄**:前向积累慢,头几周每桶 n 个位数,**scorecard 必带 n,n 小不下结论**。v1.6 建的是**标尺**,不是"现在就有漂亮命中率"。
- **红线**:真打分跑出来前,前端**绝不显示命中率数字**(沿用 V1.5/V2 红线)。
- 命中率页的**完整接入**(heatmap / 月季年 rollup / 区间聚合)仍是 V2-S4;v1.6 先产出 `scorecard.json` 工件 + md。

### S1 · 量化因子层(后端架构主体)

**新增 `py/newsletter/factors.py`**(纯函数,`snap` 的高阶变换;`snap` 已因果 → 因子天然因果):
- `compute_factors(snap: dict[str, float]) -> dict[str, FactorView]`,对 `catalog.PREDICTION_TARGET_IDS` 各算一组。
- **按 kind 分支**(关键坑:`DGS2` 是利率,`features.compute_features` 不给它 `px_vs_ma200`/`ret_*`,只有 `_level`/`_z_252`/`_chg_*`):
  - 价格/指数(纳指/黄金/广义美元):`trend=clip(px_vs_ma200/0.10,-1,1)`、`momentum=clip(ret_20/0.10,-1,1)`、`value=(rangepct_252-0.5)*2`(高=贵,均值回归取反号)、`vol=自身/VIX 的 vol z`。
  - 利率(`DGS2`):`trend=clip(chg_60bp/50,-1,1)`、`momentum=clip(chg_20bp/30,-1,1)`、`value=clip(z_252/2,-1,1)`。
- **合成**:`composite = 0.5*trend + 0.4*momentum − 0.1*value_extreme`(`value_extreme` 仅在 |value|>0.6 时计入);
  `baselineDir = up/down/flat`(死区 `|composite|<0.1`);`baselineConf = 0.5 + min(0.45, 0.5*|composite|)`。权重为初值、宜少宜稳,后按 S3 校准回调(**不做花哨优化**)。
- 波动率预测留给 features(见下),factors 读取并放进 `volForecastAnn`。

**`py/newsletter/features.py` 增 EWMA 波动率(保因果,复用现有红线单测)**:
- 加 `_vol_ewma(s, lam=0.94)`:RiskMetrics 递归 `σ²_t=λσ²_{t-1}+(1-λ)r²_t`,即 `r2.ewm(alpha=1-lam, adjust=False).mean()**0.5 * _SQRT252`(只看过去,因果)。
- 在 `compute_features` 价格/指数分支加列 `{sid}_vol_ewma`;`FEATURE_VIEW` 可选加几行(vol 组)顺带露出给 signals。

**`py/newsletter/models.py` 增契约**:
- `FactorView(_CamelModel){scores:dict[str,float], composite:float, baseline_dir(alias baselineDir):PredDir, baseline_conf(alias baselineConf):float, vol_forecast_ann(alias volForecastAnn):float}`。
- `Brief` 脊柱加 `factors: dict[str, FactorView] = Field(default_factory=dict)`(key=series_id 小写,同 `price_series` 款)。旧 brief 无此字段 → 默认空 dict,`_migrate_legacy` 不受影响。

**`py/newsletter/render.py` + `pipeline.py` 接线**:
- `render.build_factors(snap) -> dict[str, FactorView]`(薄封装 `factors.compute_factors`,key 转小写)。
- `build_brief(...)` 增 `factors` 形参(同 `signals`/`price_series` 的可选脊柱注入)。

### 6.4 新闻 → 代码特征(进预测的那条路)

`news/features.py`:从已分类新闻,对 roster 各资产代码算(point-in-time):
- **事件日历标记** `event_flag`:未来 N 个交易日内有 FOMC/CPI/NFP(代码日历)→ 让 LLM 在催化剂前**降低信心** → 改善**校准**(唯一稳的赢法,蹭 pre-FOMC drift)。
- **新闻量 z** `news_volume_z`:每资产标题数对 20 日均值的 z → 喂 **波动率预测**(波动才是可预测的;高关注≈高波动,方向上反而常反转)。
- **惊喜** `surprise = 实际 − 预期`:CPI/NFP 等有数值预期处(可硬编 consensus)→ rates/USD 唯一有 5d 方向含量的新闻类输入;**这是数据,不是情绪**。
- 净情绪 + 新颖度(近 5 日去重,Tetlock:陈旧新闻会反转)→ **仅作前端叙事 + 「新闻 vs 盘面分歧」提示**,**不进被打分预测**。

这些**代码算的**特征作为一个**受治理的数字块**进 `build_feature_block`(仅 B 臂),守「代码算数字、LLM 只解释」——避免裸情绪散文导致的信心通胀。

### 6.5 A/B 消融协议(让评估层回答「新闻值多少」)

- `predictions.csv` 幂等键 **+= `arm`**(`A`=仅价格因子;`B`=A + §6.4 新闻特征);两臂用**同一个 LLM 调用预算 + 同一份市场数据**,**唯一差异 = 新闻特征块**。
- **前向 only**:回填新闻有「发布滞后/前视」泄漏(v2 §1b 的新闻版)。**白送的泄漏检测**:若 B 在回填里漂亮、前向却平 —— 那个差距**就是污染**,证明信号是假的。
- `evaluate.py`(S3)按臂比 **skill / Brier / 校准**,block-bootstrap 给区间,**预先约定停止规则**:仅当 `skill(B)>skill(A)` 且 `Brier(B)<Brier(A)` 且区间不含 0 才算「新闻有用」。**决策不对称**:方向没升但 Brier 变差(预期失败模式)→ **直接否决**。
- 可选对照臂:新闻量 → 波动率预测,按已实现波动打分(方向多半平、波动可能正,凸显「方向 vs 波动」差异)。
