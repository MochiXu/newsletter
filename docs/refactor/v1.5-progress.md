## 1. 后端对外契约加厚(`briefs.json`)

V1 的 `briefs.json` 只有四层 + 指标表。这一阶段把"代码算好的东西"尽量暴露给前端,并让假设层变成可评估的预测。

| 增强 | 内容 | 关键代码 |
|---|---|---|
| **technical signals** | 29 条技术指标(趋势/动量/波动/利率·通胀/美元/跨资产相关/52周分位),带 `unit`(pct/pct0/bp/z/corr/yield)+ `group`;单一事实源 `features.FEATURE_VIEW` | `render.build_signals` |
| **regime 标签** | 6 个代码派生的市场状态(股票趋势/波动/曲线/实际利率/通胀预期/美元) | `regime.derive` |
| **假设层 → 预测卡** | 假设层改为对**固定 roster**(纳指 NASDAQCOM / 黄金 XAUUSD / 广义美元 DTWEXBGS / 2Y DGS2)各给且只给一条预测,带 `asset/direction/horizon/confidence/key_factors/可度量失效` | `catalog.PREDICTION_TARGETS`、`schema` |
| **metric.spark** | 每个指标最近 ~20 个真实收盘点(因果),供前端小走势线 + hover | `features.metric_spark` |
| **priceSeries** | 5 个图表资产(纳指/黄金/广义美元/US10Y/VIX)各 30 个真实价格点(带日期,因果),供前端可交互 30D 价格大图 | `features.price_series`、`render.build_price_series`、`catalog.CHART_SERIES_IDS` |
| **facts/reads 标签化 + 数字方向** | 事实层/解读层从 `str[]` 升级为 `[{tag,text,figures}]`:`tag` 主题标签(受控词表 `models.FACT_TAGS`)、`figures:[{t,dir}]` 标注正文里**有方向含义的变化数字**(供前端按 up/down/flat 上色)。LLM 在 schema 里填**扁平字符串** `figs='token|dir;...'`,`_parse_figs` 解析成结构化 figures | `models.TaggedItem/Figure`、`schema._FIGS_PROP` |
| **新闻升级** | 优质宏观源(Fed/ECB/MarketWatch/CNBC)+ 过滤噪音/无链接 + 带真实 link;fetch 与 classify 解耦(分类失败不丢新闻) | `news.py`、`pipeline` |

**通用文本规范化**(适用所有展示文本:headline / 事实 / 解读 / 影响 / 复盘 / 预测卡):
- `textnorm.normalize_text`(确定性后处理,render 落库前统一应用):全角标点 → ASCII、中文与字母数字之间盘古空格、双引号 → 单引号(避免破坏 JSON);**不拆开** `MA200 / 2s10s / 9bp / 0.27%` 等字母数字词。
- `llm/style.TEXT_STYLE`(公共 prompt 片段,简报 + 新闻共用):向 LLM 表达同样的标点/空格/引号约定,减少后处理改动量。

## 2. LLM 健壮性

| 问题 | 修复 |
|---|---|
| **DeepSeek 强制 function-calling 在复杂 schema 下结构失稳**:加了嵌套字段后,会在 facts 数组后提前闭合根对象、把 interpretation 当兄弟对象续写 → JSON 必解析失败、丢字段 | 结构化输出改用 **JSON mode**(`response_format=json_object` + 把 JSON Schema 写进 system,需含 'JSON' 字样),读 `message.content`;`max_tokens` 4096→8192 |
| figures 用"数组套对象再套数组"的深层嵌套也会触发上面的失稳 | figures 在 schema 里用**扁平字符串** `figs='token|dir;...'`,后端解析(扁平串 LLM 友好,前端契约仍是 `figures:[{t,dir}]`) |
| 新闻分类失败会把已抓到的新闻一并丢掉 | fetch / classify 解耦,分类失败降级为"未分类新闻"仍展示 |
| **Claude 经中转站对复杂 schema 偶发不回 `tool_use`、把整段 JSON 当文本吐**(```json 围栏) | `AnthropicProvider` 三路兜底(原生 tool_use → 中转站加的 OpenAI 风格 `tool_calls` → 文本里抠 JSON)+ 重试 |

## 4. 多模型生成 + 中转站接入(2026-06-23)

**动机**:单模型是「一家之言」。让多个模型各出一份解读,**展示分歧本身就是信息**(对学习型产品尤其);而「该信哪个模型」**不靠另一个 LLM 收敛**(会污染 + 洗掉分歧 + 收敛器自己也没被验证),交给 V2 的**战绩(命中率/校准)**回答。所以本阶段只做「**生成 + 落库 + 前端切换 + 代码级共识**」,不做 LLM 收敛。

- **契约**:`Brief` 拆为**脊柱(模型无关,代码算)+ `views`(按模型的六层解读)+ `consensus`**。脊柱 = metrics/signals/regime/priceSeries/news/reviews(算一次,所有模型共享);随模型变的只有 tone/headline/facts/reads/hypotheses/impacts(每模型一份 `ModelView`)。
- **provider 枚举**:`LLM_MODELS=deepseek,anthropic,openai`(逗号列表,[0]=主模型;缺 key 自动跳过;未设 = 回退单模型)。`generate_briefs` 逐模型生成,**单模型失败只跳过它**,不影响其余(优雅降级)。
- **代码级共识**:对固定 roster 各资产**纯代码投票**(多数方向 + 各方向票数 + 认同数 + 多数方向均值信心,平票=分歧/flat)。不调 LLM、抗污染(烂模型被票数稀释)。**目前等权**;V2 per-model 战绩出来后再**按校准加权**(比 LLM 收敛高级,加权入参待加)。
- **前端**:右上角**模型切换器**(>1 模型才显示,默认主模型),切换即换整张 AI BRIEF 视图;假设层上方一行**跨模型共识**;另加**信心 tooltip**(说明 confidence 是模型自评、未校准、非真实概率,仅供同期横向参考)。
- **中转站接入**(`sub.foyego.com`,跑 Claude `claude-opus-4-8` + OpenAI `gpt-5.5`;DeepSeek 直连):`AnthropicProvider` 支持 `ANTHROPIC_BASE_URL` + `ANTHROPIC_AUTH_TOKEN`(x-api-key & Bearer 双头);`_compat_url` 让 OpenAI base 填 host 自动补 `/v1/chat/completions`。**排坑见 §2 末行 + 后端 README**:每模型族用**不同 key**;base 只填 host(别加 `/v1`);Claude 文本兜底;在 Claude Code 自己 shell 里跑须 `env -u ANTHROPIC_BASE_URL`。
- **向后兼容**:旧扁平 brief(顶层 facts/reads/...)由 `Brief` 的 `model_validator` 自动迁进 `views.archive`,旧 `briefs.json` 不崩。

## 5. 当前 `briefs.json` 契约(完整)

顶层 `{model, generatedAt, briefs:[Brief]}`(`model` = 参与模型的人读标签,如 `DeepSeek + Claude + GPT`)。`Brief`:
`date / weekday / issue / time`
· 脊柱:`metrics[{key,label,value,change,kind,spark:[{date,value}]}]` · `signals[{key,label,value,unit,group}]` · `regime{}` · `priceSeries{key:[{date,value}]}`(5 资产×30 点)· `reviews[{ifThen,status,note}]` · `news[{title,source,cat,assets,dir,link}]`
· 多模型:`models[]`(模型 id,有序,[0]=主)· `views{modelId: ModelView}` · `consensus[ConsensusItem]`(≥2 模型才有)
`ModelView` = `{tone, headline, facts[{tag,text,figures:[{t,dir}]}], reads[同], hypotheses[{ifThen,invalidation,asset,direction,horizon,confidence,keyFactors:[{label,detail}]}], impacts[{asset,watch,dir,code}]}`
`KeyFactor` = `{label(短标签,chip 常显), detail(完整读数,hover)}`;LLM 以扁平串 `'label|detail'` 产出,`render._parse_key_factors` 解析(同 figs,避开数组套对象的双层嵌套)。
`ConsensusItem` = `{asset, direction, votes{up,down,flat}, n, agree, meanConfidence}`
单一事实源 = `py/newsletter/models.py`;所有展示文本经 `textnorm.normalize_text`。
